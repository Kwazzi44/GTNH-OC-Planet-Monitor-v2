# GTNH Planet Monitor V2.0

Централизованная система мониторинга мультиблоков GregTech на удалённых планетах (или в пределах одной базы) через **OpenComputers**.
Система работает напрямую с компонентами `gt_machine` по сети, без необходимости ставить отдельные компьютеры на каждой планете.

---

## 🚀 Возможности
- **Мониторинг в реальном времени:** Отслеживание статуса машин (OK, STBY, PROB).
- **Детализация:** Отображение активности (ACTIVE/INACTIVE), прогресса выполнения и возникающих проблем.
- **Удаленное управление:** Перезапуск контроллеров (программный или через Redstone).
- **Организация:** Группировка машин по планетам в удобном интерфейсе.
- **Логирование:** Запись всех событий (сбои, перезапуски) в лог.
- **Телеметрия:** Отображение заряда LSC (Лапотронного суперкристалла) и TPS сервера.

## 📁 Структура файлов
```text
/home/
├── hub/

```

## 💻 Требования к оборудованию

Для работы системы требуется следующий комплект оборудования:

### 1. Серверная стойка (Server Rack) с компьютером T3
![Внешний вид стойки](image.png)

Используемый комплект компонентов:
- **Оперативная память:** Планка памяти T3.5 — 2 шт.
- **Процессор:** Центральный процессор T3.
- **Видеокарта:** Видеокарта T3.
- **Жесткий диск:** Жёсткий диск T3.
- **Интернет-карта:** (необходима для установки и обновления).
- **Компонентная шина T3:** (если требуется подключить более 16 адаптеров, рекомендуется использовать Творческую шину).

### 2. Периферия и подключение
- **Монитор T3.**
- **Адаптеры OpenComputers:** Подключаются к контроллерам мультиблоков.
  - ⚠️ *Важно:* Адаптеры подключаются с помощью MFU. Настоятельно рекомендуется, чтобы к граням адаптера не примыкал никакой другой блок.

## 📥 Установка

Для первой установки выполните в терминале OpenComputers:
```bash
wget -q https://raw.githubusercontent.com/Kwazzi44/GTNH-OC-Planet-Monitor-v2/main/install_hub.lua
```
Затем запустите скачанный файл:
```bash
install_hub.lua
```
Скрипт скачает все файлы в `/home/hub/` и предложит создать автозапуск.

---

## ⚙️ Настройка

1. Запустите программу (или мастер настройки):
   ```bash
   /home/hub/main.lua
   ```
2. Выберите пункт **Scan New Machines**. Скрипт найдет все неподключенные адаптеры `gt_machine`.
3. Для каждой машины:
   - Введите название планеты (можно выбрать из списка подсказок или ввести свою).
   - Введите понятное имя для машины (или оставьте то имя, которое предложила программа).
4. Вы можете назначить один из адаптеров как LSC для мониторинга энергии.
5. Выйдите из сетапа с помощью меню.

## 🎮 Управление

В основном интерфейсе (`lua /home/hub/main.lua`):

| Клавиша | Действие |
| :--- | :--- |
| `↑ / ↓` | Навигация по списку |
| `Enter` | Открыть детали планеты / подтвердить |
| `B` | Назад / выход в список планет |
| `F4` | Открыть лог событий |
| `F3` | Принудительный опрос всех машин |
| `A` | Перезапустить все выключенные машины на выбранной планете |
| `T` | Переключить режим работы машины (Toggle) |
| `F5` | Обновить скрипт (если вышло обновление) |
| `Q` / `Esc` | Выход из программы |

---

## 🔄 Обновление

Если вышло обновление скрипта, нажмите `F5` в главном меню программы. Скрипт скачает только измененные файлы, не затирая вашу базу данных планет.
