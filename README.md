# GTNH-OC-Planet-Monitor

Централизованная система мониторинга мультиблоков на удалённых планетах через **OpenComputers**.  
Показывает статус машин в реальном времени, позволяет перезапускать контроллеры удалённо, логирует все события.

## Структура файлов

```
GTNH-OC-Planet-Monitor/
├── protocol.lua          — Общая библиотека протокола (нужна и Hub, и Node)
├── hub/
│   ├── main.lua          — Точка входа Hub (главная база)
│   ├── config.lua        — Настройки Hub (порт, таймауты)
│   ├── monitor.lua       — Логика опроса планет
│   ├── registry.lua      — Реестр планет (сохраняется в hub_registry.json)
│   ├── logger.lua        — Лог событий (hub_eventlog.txt)
│   └── gui.lua           — Двухуровневый GUI
└── node/
    ├── main.lua          — Точка входа Node (планета)
    ├── node_config.lua   — Конфиг планеты (только имя!)
    └── machines.lua      — Авто-детект GT-адаптеров, restart
```

## Требования к оборудованию

### Hub-компьютер (главная база)
- OC Computer T2+
- GPU T2+ и Screen
- **Modem Card** (для связи через P2P туннели)

### Node-компьютер (каждая планета)
- OC Computer T1+ (GUI не нужен)
- **Modem Card** (для связи через P2P туннель)
- OC **Adapter** блоки рядом с каждым контроллером мультиблока (соединены кабелем с Node)
- (Опционально) **Redstone I/O** блок для fallback-restart

### Сеть
- **OC P2P туннели** соединяют все Node-компьютеры с Hub через единую OC-сеть
- Адаптеры подключены **только локально** к своему Node (кабелем), Hub их не видит напрямую

## Установка

### 1. Hub (главная база)

Скопируй на компьютер Hub:
```
protocol.lua   → /protocol.lua   (или /lib/protocol.lua)
hub/config.lua → /hub/config.lua
hub/logger.lua → /hub/logger.lua
hub/registry.lua → /hub/registry.lua
hub/monitor.lua → /hub/monitor.lua
hub/gui.lua    → /hub/gui.lua
hub/main.lua   → /hub/main.lua
```

Запуск:
```
lua /hub/main.lua
```

### 2. Node (каждая планета)

Скопируй на компьютер планеты:
```
protocol.lua       → /protocol.lua
node/machines.lua  → /node/machines.lua
node/main.lua      → /node/main.lua
node/node_config.lua → /node/node_config.lua
```

Отредактируй `node_config.lua` — впиши имя планеты:
```lua
config.planet_name = "Ceres"   -- <- только это нужно изменить
```

Запуск:
```
lua /node/main.lua
```

Для автозапуска создай `/home/autorun.lua`:
```lua
shell.execute("/node/main.lua")
```

## GUI — управление

### Главный экран (список планет)
| Клавиша | Действие |
|---------|---------|
| `↑ / ↓` | Навигация по планетам |
| `ENTER` | Открыть детали планеты |
| `A` | Restart All машин на выбранной планете |
| `R` | Принудительный Refresh (PING всех) |
| `L` | Открыть лог событий |
| `Q` | Выход |

### Экран деталей планеты
| Клавиша | Действие |
|---------|---------|
| `↑ / ↓` | Навигация по машинам |
| `ENTER` | Restart выбранной машины |
| `A` | Restart All выключенных машин |
| `S` | Scan — пересканировать адаптеры на планете |
| `B` | Назад |

### Экран лога
| Клавиша | Действие |
|---------|---------|
| `↑ / ↓` | Прокрутка |
| `Home` | В начало |
| `End` | В конец (последние события) |
| `B` | Назад |

## Статусы планет

| Иконка | Статус | Значение |
|--------|--------|---------|
| OK | 🟢 | Все машины активны |
| PARTIAL | 🟡 | Есть выключенные машины — можно перезапустить |
| RING DOWN | 🔴 | Node не отвечает — квантовое кольцо выключено, **нужно ручное вмешательство** |
| UNKNOWN | ⚫ | Данных ещё нет |

## Добавление новой планеты

1. Поставь компьютер на планете
2. Подключи P2P туннель (к Hub-сети)
3. Подключи GT Adapter'ы к мультиблокам (OC кабелями к Node-компьютеру)
4. Скопируй файлы Node, заполни `planet_name` в `node_config.lua`
5. Запусти `main.lua` — Hub подхватит планету автоматически ✅

## Добавление нового мультиблока на существующую планету

1. Поставь GT Adapter рядом с новым мультиблоком
2. Подключи кабелем к Node-компьютеру
3. На Hub: перейди в детали планеты → нажми `[S] Scan`
4. Node пересканирует адаптеры, Hub обновит список ✅

## Технические заметки

- **GT Adapter restart**: пробуется `setWorkAllowed(true)`, `enable()`, `setEnabled(true)` поочерёдно
- **Redstone fallback**: если ни один метод адаптера не сработал — подаётся импульс через `redstone.setOutput`
- **Реестр**: сохраняется в `/hub_registry.json`, переживает перезапуск Hub
- **Лог**: пишется в `/hub_eventlog.txt`, хранит последние 500 строк
- **Авто-скан на Node**: каждые 60 секунд Node полностью пересканирует адаптеры
